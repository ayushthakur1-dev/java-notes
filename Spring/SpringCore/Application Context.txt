The application context is the object that creates, configures, wires and manages all Spring beans.

Responsibilities of Application Context: 
bean creation
dependency injection
lifecycle of bean
scopes
loading properties
publishing events
 
without it spring doesn't exist

At startup: 
1. scans configuration
2. creates beans
3. resolved dependencies 
4. puts everything into the context
when something needs a bean -> it asks the context

But then how it is different from bean factory? 
ApplicationContext extends BeanFactory, so it can do everything it can do plus more


How it gets created: 
In modern apps (especially spring boot) you rarely create it manually 
Boot does it when the application starts: SpringApplication.run(App.class, args);

ways to access the application context: 
most of the time you should not need to, dependency injection should be enough, but sometimes you need manual access

Method 1: inject it
@Autowired
private ApplicationContext context;

now you can: 
PaymentService ps = context.getBean(PaymentService.class);

Method 2: Implement ApplicationContextAware:
spring will call you and provide it

@Component
class ContextHolder implements ApplicationContextAware {
	private static ApplicationContext context;
	
	@Override
	public void setApplicationContext(ApplicationContext ctx) {
		context = ctx;
	}

	public static ApplicationContext getContext() {
		return context;
	}
}

Method 3: From main method
ApplicationContext ctx = SpringApplication.run(App.class, args);

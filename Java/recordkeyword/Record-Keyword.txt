Passing immutable data between object is one of the most common, but mundane(non-exciting) task in many java applications.

Prior to Java14 this requires the creation of a class with boilerplate fields and methods, which where susceptible(liable) to trivial mistakes and muddled(unordered) intentions, but now we can use 'record' to remedy these problems

commonly we write classes to hold data, in many cases the data is immutable, since immutability ensures validity of the data without synchronization

to accomplish this, we create fields like: 
private, final for each piece of data
getters
equals method
hashCode methods
toString methods

While this accomplish our goals there are two problems to it 
1. There is a lot of boiler plate code
2. the extra code obscures that our class is simply a data class

Now we can replace our repetitious data classes with records
Records are immutable data classes that require only the type and name of fields, the equals, hashCode, and toString methods, as well as private fields final fields and public constructor are generated by the java compiler

But, records are final in Java hence they cannot be extended by any other class
Also they already java.lang.Record class and hence cannot extend any other class but they can implement interface
